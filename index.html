Comando básicos Github

Esse tutorial tem o intuito de apresentar comandos básico e úteis para utilização no Github e versionamento de projetos.

Descrição: O Git é um sistema de controle de revisão distribuído, rápido, escalável e com um conjunto de comandos incomumente rico que oferece operações de alto nível e acesso completo aos seus recursos.

Abrindo projeto com Git Bash

Primeiramente vamos abrir a pasta do nosso projeto para podermos iniciar o repositório. Após abrir a posta do projeto clique com o botão direito na área e em seguida clique em Git Bash Here para abrir o terminal.

Figura 1: Acessando Git Bash na pasta do projeto

Iniciando Repositório no projeto local

Comando: git init

Note que após executar o comando ele iniciou um repositório vázio no diretório local.

Esse comando cria um repositório Git vazio, basicamente um diretório .git com subdiretórios para os arquivos objects, refs/heads, refs/tags e arquivos modelo.

Figura 2: Executando comando para iniciar repositório local

Adicionando os arquivos no repositório local

Comando: git add *

Este comando atualiza o índice utilizando o conteúdo atual encontrado na árvore de trabalho para preparar o conteúdo para o próximo commit. Em geral ele adiciona o conteúdo atual dos caminhos existentes como um todo, mas com algumas opções ele também pode ser utilizado para adicionar o conteúdo com apenas a parte das alterações aplicadas nos arquivos da árvore de trabalho ou remover os caminhos que não existam mais na árvore de trabalho.

Figura 3: Adicionando os arquivos no repositório

Executando Commit no repositório

Comando: git commint -m "Mensagem do commit"

Cria um novo commit que tenha todos os conteúdos atuais do índice e a mensagem informada no registro log descrevendo as alterações.

Figura 3: Executando commit no repositório

Monitorando o status do repositório

Comando: git status

Exibe os caminhos que têm diferenças entre o arquivo do índice e o commit atual no HEAD, os caminhos que têm diferenças entre a árvore de trabalho e o arquivo do índice, os caminhos na árvore de trabalho que não são rastreados pelo Git.

Figura 5: mostramos o status do repositório com erros, pois houve alteração nos arquivos dos repositório e para corrigir esse erro execute o comando git add * para que o status seja corrigido.

Figura 5: Status exibido com problemas.

Listando as configurações do git

Comando: git config --list

Lista todas as variáveis definidas no arquivo de configuração, junto com seus valores.

Figura 6: Listando as configuração do git

Configurando as credenciais do git

Comando: git config --global (credencial que vamos configurar)

Com este comando estaremos configurando as crendenciais de acesso ao github, iniciaremos configurando a credencial user.name, em seguida a user.email e por último a user.nickname.

git config --global user.name "(Informe o nome desejado)"

git config --global user.email "(Informe o e-mail de cadastrado do Github)"

git config --global user.nickname "(Informe o usuário cadastrado do Github)"

Observação: as informações das crendenciais devem estar entre "".

Figura 8: Configurando crendencial e-mail

Corrigindo as credenciais do git

Comando: git config --global --unset user.name

Com este comando estaremos removendo a crendencial que foi configurada errada.

Ao executar o comando git config --list, você verá que a crendial user.nome não aparecerá na lista, assim você poderá adicionar a crendial novamente a lista.

Mas antes precisamos entender o que é a extensão .md - Markdown

O que é Markdown?

De acordo com a Wikipédia:

Markdown é uma linguagem de marcação leve com sintaxe de formatação de texto simples projetada para que ela possa ser convertida em HTML e muitos outros formatos usando uma ferramenta com o mesmo nome. Markdown é usado frequentemente para formatar arquivos readme, para escrever mensagens em fóruns de discussão on-line e para criar texto rico usando um editor de texto simples.

O Markdown não exites um padrão claramente definido para ele, onde o levou à possuir diferentes fornecedores que escrevem suas próprias variantes do idioma para corrigir falhas e adicionar recursos faltantes. Temos uma lista de versões do Markdown disponível aqui.

Porque usar o Markdown?

  • FÁCIL: A sintaxe é tão fácil que você pode aprender em um minuto ou dois, em seguida, escreva sem perceber nada estranho ou nerd.
  • RÁPIDO: Ele economiza tempo em comparação com outros tipos de arquivos / formatos de texto. Isso ajuda a aumentar a produtividade e os fluxos de trabalho do escritor.
  • LIMPO: Tanto a sintaxe como a saída são limpas, sem confusão com nossos olhos e simples de gerenciar.
  • FLEXÍVEL: Com apenas algumas configurações, o seu texto será traduzido atravessando qualquer plataforma lá fora, editável em qualquer software de edição de texto e conversível para uma ampla variedade de formatos.

Em resumo, os usuários comuns acharão útil em todos os casos, especialmente quando você precisar de algo melhor que o texto simples, mas menos funcional do que o Microsoft Word.

Para desenvolvedores, Se você é preguiçoso para escrever código HTML, você vai adorar o markdown. Além disso, Github e muitos sites favorecem o markdown para o arquivo readme de projetos. Isso significa que você vai encontrar o markdown em sua vida de uma forma ou de outra.


Ferramentas Markdown

Qualquer editor pode ser usado para editar o markdown. No entanto, existem algumas ferramentas que podem ser úteis para você quando se trata de editar markdown.

  • Stackedit: Ferramenta on-line, basta digitar o texto normalmente e depois efetuar a formatação. Você não precisa saber a sintaxe.
  • Dillinger: Ferramenta on-line, suporte em exibição ao vivo (tela dividida) e exportação para html. Nada muito especial, mas muito limpo e acessível.
  • Typora: Disponível para Mac e Windows, livre de distração, vista ao vivo sem parecer, empacotada com muitas outras coisas como Imagens, Listas, Tabelas, Cercas de Código, Blocos de Matemática, YAML, Front Matters, Toc.
  • Minimalist Markdown: Aplicativo do Google Chrome. Funciona em todos os lugares se você tiver o Chrome instalado.
  • Macdown: O Melhor para Mac.
  • MarkdownPad: O Melhor para Windows.
  • Remarkable: O Melhor para Linux.
  • GITBOOK: O GitBook é uma ferramenta de publicação moderna. Facilitando a escrita e a colaboração. Ambos suportam a Markdown e têm uma estreita relação com o amado Github.


Sintaxes do Markdown

Você pode consultar toda a sintaxe do Markdown acessando aqui. Abaixo listaremos as sintaxes básicas, apenas para exemplificar neste tutorial.

Formato Sintaxe Exemplo
Itálico *Texto* Meu texto em itálico
Negrito **Bold** Meu texto em negrito
Links inline [texto](url aqui) Meu link
Imagens ![Legenda](url da img) Minha imagem
Link+imagens [![Legenda](url da img)](url para a pagina)] Clique aqui
Quebras de linha Double space + enter
Listas não ordenadas * Item1 *Item 2
  • item1
  • item2
Listas Ordenadas 1. Item a 2. Item b
  1. item a
  2. item b
Listas mistas 1. Item 1 * item 1a
  1. item a
  • item 1
Citação > Texto citado
Minha citação
Código `Insira o código` cout<<"Hello world";
Bloco de código/ Destaque de Sintaxe ```Insira o código```
Títulos #, ##, ###, ####, #####, ###### (from h1 to h6)

Isso é um título h3
Riscado ~Insira o texto aqui~ Meu texto riscado
Tabelas | Tables | Are | Cool | |----------|:-------------:|------:| | col 1 is| left-aligned | $1600 |


Notas úteis

  • O Markdown nos permite utilizar escapes de barra invertida para gerar caracteres literais que teria um significado especial na sintaxe de formatação da Markdown.

    Assim? \ * Este \ * não é mais itálico, mas é cercado por asteriscos literais.


  • Os vídeos do Youtube requerem uma atenção especial

    Eles não podem ser adicionados diretamente, mas você pode adicionar uma imagem com um link para o vídeo como este:
    <a href="http://www.youtube.com/watch?feature=player_embedded&v=YOUTUBE_VIDEO_ID_HERE" target="_blank"> <img src="http://img.youtube.com/vi/YOUTUBE_VIDEO_ID_HERE/0.jpg" alt="Texto ALT da imagem aqui" width="240" height="180" border="10" /> </a>


  • Markdown suporta Emoji 😆 😆 😘 😇 💚 ( obter alguns emojis aqui ).
  • Você pode usar a tag <br/> para forçar uma quebra de linha.
  • O espaço duplo, em seguida, se você quiser fazer uma nova linha se houver problemas para criar novas linhas.
  • Ver não é tão bom como praticar. Você pode criar um arquivo de markdown para você praticar ou fazê-lo online aqui.
  • Qualquer URL (Como http://www.github.com/ ) será automaticamente convertido em um link clicável.
  • O suporte à tabela Markdown é projetado para lidar com a maioria das tabelas para a maioria das pessoas; não cobre todas as tabelas para todas as pessoas. Se você precisa de tabelas complexas, você precisará criá-las manualmente ou com uma ferramenta especificamente projetada para o formato de saída.
